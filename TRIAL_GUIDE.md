# Annotation to Prompt 试用说明 v0.1

## 这是什么

Annotation to Prompt 是一个 ChatGPT 网页版批注扩展。

它解决的问题是：当 ChatGPT / Agent 生成一大段内容后，用户不想再手写一长串“针对第 1 点、第 2 点、第 3 点分别修改什么”，而是可以直接在原回答上选中文段、添加批注，再把这些批注自动整理成下一轮 prompt。

一句话：

```text
把用户对 Agent 输出的零散判断，转成结构化的下一轮修改指令。
```

## 适用场景

适合：

- 长回答校准：对 ChatGPT 的长回答逐段批注，要求它改口径、补证据、删掉、展开。
- 工作流设计：讨论一个方案时，直接标出哪些模块不准确、哪些要重写、哪些要新增任务。
- PhD / 研究兴趣梳理：对 research memo、faculty fit、文献搜索工作流进行结构化反馈。
- 文档草稿修改：对 SOP、CV bullet、汇报文档、产品说明做逐段批注。
- Agent 协作调教：把“人类判断”低摩擦回流给下一轮 Agent。

不适合：

- 自动代替你发送消息。
- 批注网页上所有内容。
- 处理多人协作文档。
- 保存长期记忆或自动写入知识库。
- 移动端 ChatGPT App。
- 高度敏感、涉密或不适合放进 ChatGPT 的内容。

## 当前功能

- 选中 ChatGPT assistant 已生成回复中的文字。
- 点击浮动「批注」按钮，或右键选择「添加批注」。
- 在弹窗中输入最多 1000 字批注。
- 保存后，原文会出现高亮下划线和批注序号。
- 点击序号可以编辑或删除批注。
- 删除批注后，序号会自动重排。
- 右下角控制条支持：
  - 同步到输入框
  - 复制批注
  - 清空批注
- 批注会编译成 `【批注汇总】...【/批注汇总】` 格式，用户手动发送给 ChatGPT。

## 安装方式

1. 下载或 clone 本项目。
2. 打开 Chrome / Edge。
3. 地址栏输入：

```text
chrome://extensions/
```

4. 打开右上角「开发者模式」。
5. 点击「加载已解压的扩展程序」。
6. 选择项目目录：

```text
annotation-to-prompt
```

7. 打开或刷新 ChatGPT 网页版：

```text
https://chatgpt.com/
```

## 使用流程

1. 等 ChatGPT 完成一段回复。
2. 选中你想批注的文字。
3. 点击浮动「批注」按钮。
4. 输入批注内容。
5. 点击「暂存批注」或按 `Ctrl+Enter`。
6. 重复批注多个片段。
7. 点击右下角「同步到输入框」或「复制批注」。
8. 检查输入框内容。
9. 手动点击 ChatGPT 发送按钮。

生成的 prompt 类似：

```text
【批注汇总】
请基于以下批注处理上一轮输出，保留未被批注部分的主体结构：

1. 针对「关键词不一致」：
用户批注：这个论断不全面，需要加入概念层级错配。

2. 针对「OpenAlex」：
用户批注：这个场景可能不适用，请先评估 API 可用性和引用质量。
【/批注汇总】
```

## 功能边界

触发边界：

- 只有用户主动选中 assistant 回复文本时才触发。
- 输入框、按钮、系统 UI、用户自己的消息里选中文本不会作为主要支持范围。

发送边界：

- 插件不会自动点击发送。
- 插件只把批注整理到输入框，最终是否发送由用户决定。

缓存边界：

- 批注存储在浏览器本地 `chrome.storage.local`。
- 已保存的 ChatGPT 会话按 `/c/{conversationId}` 隔离。
- 新建未保存会话按当前浏览器 tab 临时隔离。
- 刷新页面后会恢复高亮和控制条，但不会自动写入输入框。

数据边界：

- 第一版没有后端。
- 不上传批注。
- 不读取账号信息。
- 不拦截网络请求。

## 已知限制

- ChatGPT 页面结构变化可能导致按钮、输入框同步或高亮定位失效。
- 重复文本很多时，高亮可能定位到不完全符合预期的位置。
- 跨多个复杂 DOM 节点的长选择，仍可能有高亮失败风险。
- 输入框自动同步不稳定时，请使用「复制批注」手动粘贴。
- 当前没有批注标签功能，只有自由文本批注。
- 当前没有导出 JSON / Markdown 功能。

## 安全提醒

- 不要把不适合发送给 ChatGPT 的涉密内容批注后发送。
- 如果批注包含工作内部信息，请先自行脱敏。
- 如果你只想保留本地批注，不要点击「同步到输入框」或发送。
- 如果测试完毕，建议点击「清空」清理当前会话批注。

## 当前版本目标

v0.1 的目标不是做完整产品，而是验证这个核心闭环是否成立：

```text
选中 → 批注 → 高亮 → 编译 prompt → 手动发送
```

如果这个闭环足够顺滑，后续再逐步加入：

- 批注标签
- 一键导出
- 多平台适配
- 批注转任务
- 批注转长期偏好
- 更稳的文本锚点系统
